# 05 学习路径

**类别：** course　**版本：** v0.1　**生效日期：** 2026-07-15　**状态：** active

## 零基础用户

建议每周投入 5 至 7 小时：先完成课程内容，再完成当周最小作业。第一周只要求选定一个真实或模拟业务场景，不要求一次做出完整系统。

## 转岗用户（设计、运营、咨询等）

建议优先把已有行业经验转成用户问题和业务资料，再完成 RAG 客服项目。作品集重点应说明：你定义了什么问题、为什么这样设计知识库、如何验证回答质量。

## 在职用户

建议每周固定两个学习时段，每次 2 至 3 小时；以“完成一个可运行的最小版本”为优先，不追求在第一周就完成微调、Agent 和复杂部署。

## 推荐顺序

1. 先写项目范围和评测问题；
2. 再整理知识库资料并做托管 RAG 基线；
3. 根据失败案例优化提示词和检索；
4. 最后再实现自建 RAG、部署和可选微调。

课程不承诺零基础用户可在六周内达到某一岗位录用标准；实际效果取决于投入时间、原有经验和项目完成质量。
